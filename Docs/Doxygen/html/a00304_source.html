3 # Test whether a client sends a correct PUBLISH to a topic with QoS 1 and responds to a delay.
4 
5 # The client should connect to port 1888 with keepalive=60, clean session set,
6 # and client id publish-qos1-test
7 # The test will send a CONNACK message to the client with rc=0. Upon receiving
8 # the CONNACK the client should verify that rc==0. If not, it should exit with
9 # return code=1.
10 # On a successful CONNACK, the client should send a PUBLISH message with topic
11 # "pub/qos1/test", payload "message" and QoS=1.
12 # The test will not respond to the first PUBLISH message, so the client must
13 # resend the PUBLISH message with dup=1. Note that to keep test durations low, a
14 # message retry timeout of less than 10 seconds is required for this test.
15 # On receiving the second PUBLISH message, the test will send the correct
16 # PUBACK response. On receiving the correct PUBACK response, the client should
17 # send a DISCONNECT message.
