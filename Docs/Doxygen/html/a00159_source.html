108 /*
109  * Topic: Threads
110  * libmosquitto provides thread safe operation, with the exception of
111  * <mosquitto_lib_init> which is not thread safe.
112  *
113  * If your application uses threads you must use <mosquitto_threaded_set> to
114  * tell the library this is the case, otherwise it makes some optimisations
115  * for the single threaded case that may result in unexpected behaviour for
116  * the multi threaded case.
117  */
118 /***************************************************
119  * Important note
120  *
121  * The following functions that deal with network operations will return
122  * MOSQ_ERR_SUCCESS on success, but this does not mean that the operation has
123  * taken place. An attempt will be made to write the network data, but if the
124  * socket is not available for writing at that time then the packet will not be
125  * sent. To ensure the packet is sent, call mosquitto_loop() (which must also
126  * be called to process incoming network data).
127  * This is especially important when disconnecting a client that has a will. If
128  * the broker does not receive the DISCONNECT command, it will assume that the
129  * client has disconnected unexpectedly and send the will.
130  *
131  * mosquitto_connect()
132  * mosquitto_disconnect()
133  * mosquitto_subscribe()
134  * mosquitto_unsubscribe()
135  * mosquitto_publish()
136  ***************************************************/
